Running the kuuzuki CLI starts it for the current directory.

```bash
kuuzuki
```

Or you can start it for a specific working directory.

```bash
kuuzuki /path/to/project
```

---

## Commands

The kuuzuki CLI also has the following commands.

---

### run

Run kuuzuki in non-interactive mode by passing a prompt directly.

```bash
kuuzuki run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "kuuzuki run"
kuuzuki run Explain the use of context in Go
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--share`    |       | Share the session                          |
| `--model`    | `-m`  | Model to use in the form of provider/model |

---

### auth

Command to manage credentials and login for providers.

```bash
kuuzuki auth [command]
```

---

#### login

Logs you into a provider and saves them in the credentials file in `~/.local/share/kuuzuki/auth.json`.

```bash
kuuzuki auth login
```

When kuuzuki starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
kuuzuki auth list
```

Or the short version.

```bash
kuuzuki auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
kuuzuki auth logout
```

---

### upgrade

Updates kuuzuki to the latest version or a specific version.

```bash
kuuzuki upgrade [target]
```

To upgrade to the latest version.

```bash
kuuzuki upgrade
```

To upgrade to a specific version.

```bash
kuuzuki upgrade v0.1.48
```

---

## Flags

The kuuzuki CLI takes the following flags.

| Flag           | Short | Description          |
| -------------- | ----- | -------------------- |
| `--help`       | `-h`  | Display help         |
| `--version`    |       | Print version number |
| `--print-logs` |       | Print logs to stderr |
| `--prompt`     | `-p`  | Prompt to use        |
| `--model`      | `-m`  | Model to use in the form of provider/model |
| `--mode`       |       | Mode to use          |
